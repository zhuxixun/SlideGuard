# SlideGuard 响应式设计源

`pages/` 中的每个 HTML 都是可独立打开的页面设计源。页面以浏览器最大化使用为默认场景，不限定为 1600×900。

## 布局规则

- 应用根容器始终填满浏览器可视区域：`100vw × 100vh`。
- 页面直接填满浏览器内容区，不模拟桌面窗口标题栏。
- 左侧导航保持稳定宽度；主内容区自动填满剩余宽度。
- 幻灯片预览、详情面板、卡片、表格和空状态根据可视区域重新排布，不对整页或文字进行非等比缩放。
- 视口较小时缩小侧栏、页面留白和详情面板宽度；必要时由主内容区滚动。
- 最小设计视口为 1180×700；低于该尺寸时保留最小布局宽高并允许浏览器滚动。

## 验证尺寸

- 1366×768
- 1600×900
- 1920×1080

`pages-1600x900/` 中的 PNG 仅是静态视觉快照，不是开发时的固定分辨率约束。实现应以 HTML/CSS 中的响应式布局规则为准。
